# Beyond Work DB-First Data Policy

## 핵심 원칙

Beyond Work의 플래너 데이터 원본은 Supabase DB의 `planner_states.state`입니다. 브라우저의 `localStorage`, 아이폰 홈화면 바로가기 캐시, 아이패드/맥 브라우저에 남은 값은 화면을 빠르게 열기 위한 임시 표시 캐시일 뿐 원본이 아닙니다.

앱을 열면 로그인 세션을 확인한 뒤 `/api/state`를 통해 DB를 읽고, 화면은 DB에서 받은 내용으로 정렬되어야 합니다. 단, 브라우저가 이미 입력을 받아 `dirty` 상태이고 그 변경 시간이 DB보다 최신이면, 그 값은 “기기 기준 데이터”가 아니라 **DB에 아직 올라가지 못한 저장 대기 변경분**으로 취급하여 먼저 DB에 저장합니다. 어느 기기에서 작성하더라도 다음에 다른 기기에서 열면 같은 DB 내용을 보게 하는 것이 기본 동작입니다.

## 작업 중 사용하지 않을 표현

사용자 화면과 작업 지시에서 아래 표현은 되도록 사용하지 않습니다.

- 동기화
- 수동 동기화
- 기기 데이터
- 로컬 데이터가 최신
- 이 기기 기준
- 백업/복원

대신 아래 표현을 사용합니다.

- 저장됨
- 저장 중
- 저장 실패
- DB에서 불러오는 중
- 내보내기
- 가져오기
- 서버 저장 보호

`내보내기/가져오기`는 안전장치일 뿐, 평상시 저장 흐름의 중심이 아닙니다.

## 앱 부팅 규칙

1. 로그인된 사용자는 앱 시작 시 반드시 `/api/state`를 먼저 확인합니다.
2. DB row가 있고 DB가 더 최신이면 DB row를 화면에 반영합니다.
3. DB row가 있으나 브라우저의 `dirty` 변경분이 DB보다 최신이면, 화면을 덮어쓰지 않고 해당 변경분을 `/api/state`로 다시 저장합니다.
4. DB row가 없으면 새 플래너로 시작합니다.
5. DB row가 없는 상태에서 브라우저에 남은 캐시를 자동으로 DB에 올리지 않습니다.
6. DB 연결 실패 상태에서는 저장 상태를 숨기지 않고 `저장 연결 실패` 또는 `다시 로그인 필요`를 표시합니다.

## 저장 규칙

1. 사용자가 내용을 입력하면 `saveState()`를 통해 로컬 화면 상태를 갱신하고 `/api/state`로 저장 요청합니다.
2. 서버 저장 전후에는 `updatedAt`, `lastSavedAt`, `dirty` 상태를 기록합니다.
3. 의미 있는 기존 데이터가 있는데 새 저장 요청의 데이터량이 급격히 줄어들면 API와 DB 트리거가 저장을 차단합니다.
4. 빈 플래너나 일부만 남은 상태가 기존 DB를 덮어쓰지 않도록 서버와 DB 양쪽에서 보호합니다.

## 개발 가이드

새 기능을 만들 때는 다음 순서로 확인합니다.

1. 새 데이터가 `state` 안의 명확한 위치에 들어가는가?
2. 저장 후 `/api/state`를 다시 불러오면 같은 데이터가 보이는가?
3. 다른 브라우저나 기기에서 로그인해도 같은 데이터가 보이는가?
4. `localStorage`만 보고 화면을 구성하는 새 경로가 생기지 않았는가?
5. 빈 값, 초기값, 마이그레이션 코드가 기존 DB 데이터를 지우지 않는가?

`localStorage`를 사용할 수 있는 경우는 화면 모드, 접힘 상태, 보안 블라인드 시간, 온보딩 접힘 여부처럼 계정 데이터가 아닌 UI 편의 설정에 한정합니다.

## 재발 방지 체크리스트

- 앱 시작 시 DB row가 있으면 항상 DB 값을 화면에 반영한다.
- 단, `dirty` 상태의 브라우저 변경분이 DB보다 최신이면 DB로 재저장하고, 서버의 과거값으로 덮어쓰지 않는다.
- DB row가 없을 때 로컬 캐시를 자동 업로드하지 않는다.
- destructive overwrite guard를 API와 DB에 유지한다.
- 사용자 화면에는 “동기화” 대신 “저장됨/저장 실패”만 표시한다.
- 구조 변경 전에는 Supabase 상태를 내보내기로 저장하고, 필요 시 SQL 백업 또는 Supabase 백업 정책을 확인한다.
- 복구 안내 문서는 `docs/data-loss-restore-2026-07-07.md`와 `docs/all-member-recovery-2026-07-07.md`에 유지한다.
