# Beyond Work 앱 온보딩 구성안

## 목적
처음 로그인한 사용자가 복잡한 기능 설명 없이, 3분 안에 오늘의 실행 화면을 완성하도록 돕습니다.

## 원칙
- 첫 사용자는 모든 기능을 배우지 않아도 됩니다.
- 앱의 첫 성공 경험은 “오늘의 우선업무 3개와 시간표 1개를 작성하는 것”입니다.
- 사용자 정보 입력은 선택이지만, AI 코칭 품질이 좋아진다는 이익을 명확히 알려줍니다.

## 온보딩 단계

### 1단계. 사용자 정보
화면 문구:
“AI가 더 정확히 돕도록 역할, 목표, 건강 리듬을 간단히 알려주세요.”

입력 항목:
- 직업/역할
- 핵심 목표
- 현재 과제
- 에너지 시간대
- 건강 상태 또는 운동 목표

버튼:
- 간단히 입력하고 시작
- 나중에 하기

### 2단계. 오늘의 우선업무 3개
화면 문구:
“오늘 반드시 의미 있는 결과를 만드는 업무 3개를 적어보세요.”

사용자 행동:
- A 업무 1개
- B 업무 1개
- C 업무 1개

보조 안내:
“A는 오늘의 핵심 결과, B는 중요한 보조 업무, C는 가능하면 처리할 일입니다.”

### 3단계. 시간표에 A업무 배치
화면 문구:
“중요한 일은 시간표 위에 놓일 때 실행됩니다.”

사용자 행동:
- 30분/1시간 선택
- A업무를 오전 또는 집중 시간대에 배치

### 4단계. AI 추천 눌러보기
화면 문구:
“AI 추천은 목표, 이월업무, 일정 상태를 보고 오늘 넣을 만한 일을 제안합니다.”

사용자 행동:
- 우선업무 AI 추천 클릭
- 추천 1개 선택

### 5단계. 오늘의 코칭 확인
화면 문구:
“말풍선은 오늘의 스마트 비서입니다. 하루의 과밀, 빈 시간, 건강 리듬을 함께 봅니다.”

사용자 행동:
- 말풍선 클릭
- 오늘의 코칭 확인

## 빈 상태 안내 문구
- 우선업무 없음: “오늘의 첫 승리를 만들 업무 하나를 적어보세요.”
- 시간표 없음: “중요업무 하나를 시간표에 고정하면 실행 확률이 올라갑니다.”
- 사용자 정보 없음: “역할과 목표를 적으면 AI 코칭이 더 개인화됩니다.”
- 반복업무 없음: “자주 반복되는 일은 요일을 선택해 자동으로 올릴 수 있습니다.”

## UI 배치 제안
- 온보딩은 전체 화면 모달보다 하단 시트 또는 중앙 작은 카드가 좋습니다.
- 모바일에서는 한 화면에 질문 하나만 보여줍니다.
- PC와 iPad에서는 왼쪽에 설명, 오른쪽에 입력칸을 둡니다.
- 모든 단계에는 “건너뛰기”가 있어야 합니다.
- 마지막 단계는 반드시 일간 일정 화면으로 돌아가야 합니다.

## 단계별 버튼 문구
- 1단계: “정보 입력하고 AI 맞춤 설정”
- 2단계: “오늘 업무 입력”
- 3단계: “시간표에 배치”
- 4단계: “AI 추천 받아보기”
- 5단계: “오늘 시작”

## 온보딩 완료 후 자동 세팅
- 일간 일정 화면으로 이동
- CEO mode 사용자는 우선업무/시간별 일정이 먼저 보이게 유지
- 말풍선 코칭 배지 표시
- 사용자 정보가 비어 있으면 설정 메뉴에 작은 안내 배지 표시

## 개발 구현 체크리스트
- 첫 로그인 여부 저장
- 온보딩 완료 여부 저장
- 나중에 하기를 눌러도 설정에서 다시 볼 수 있게 하기
- 모바일 화면에서 버튼이 하단에 잘리지 않도록 safe-area 적용
- 입력한 사용자 정보가 AI 코칭 컨텍스트에 즉시 반영되는지 확인
