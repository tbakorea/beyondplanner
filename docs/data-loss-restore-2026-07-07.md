## 추정 원인

최근 업그레이드 자체가 직접 데이터를 삭제한 정황보다는, 빈 로컬 캐시가 서버 데이터보다 최신으로 판단되어 서버 DB를 덮어쓴 가능성이 큽니다.

문제가 될 수 있는 흐름은 다음과 같습니다.

1. 특정 기기 또는 브라우저에서 로컬 플래너 상태가 빈 상태로 시작됨
2. 로컬 메타데이터의 `updatedAt`이 서버의 `updatedAt`보다 최신으로 판단됨
3. 앱이 로컬 상태를 최신으로 보고 서버에 저장함
4. 결과적으로 서버 DB의 기존 내용이 빈 상태로 대체됨

## 재발 방지 원칙

1. 서버에 기존 데이터가 있는데 로컬 데이터가 비어 있으면 로컬 값을 서버에 덮어쓰지 않습니다.
2. 로컬 데이터가 서버보다 최신이어도, 로컬 내용이 비어 있거나 현저히 적으면 사용자 확인 없이 서버 저장을 막습니다.
3. 서버 저장 전 자동 스냅샷을 남깁니다.
4. 복원 가능한 JSON 백업과 열람용 Excel 백업을 분리합니다.
5. 주요 마이그레이션이나 앱설정 구조 변경 전에는 서버 상태를 먼저 백업합니다.

## 다음 코드 보완 권장
